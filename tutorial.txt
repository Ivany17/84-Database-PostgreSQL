demons - это программы, которые работают в фоновом режиме без участия пользователей
services - это программы, которые работают в фоновом режиме под управлением пользователей

Данные (вычислительная техника) - поддающеееся многократной интерпритации представление информации в формализованном виде, пригодном для передачи, связи, или обработки (ISO/IEC 2382-1:1993).
Сереализация - процесс преобразования данных из читаемого вида в вид передачи информации
Десереализация - обратный процесс
Система управления базами данных (СУБД) - это програмный комплекс, обеспечивающий централизованное хранение данных и предоставляющий приложениям услуги по обработке данных.
База данных (БД) - совокупность данных, хранимых под управлением СУБД.

Требования к СУБД:
    - разделение программ и данных;
    - высокоуровневый язык запросов;
    - целостность;
    - согласованность;
    - отказоустойчивость;
    - защита и разграничение доступа.
    
Архитектура ANSI/SPARC
    1. Внешний уровень.
    2. Концептуальный уровень.
    3. Физический уровень.

1. Внешняя схема (приложение) - содержит описание данных в таком виде, в котором они будут использоваться приложением (отдельно для каждого приложения).
2. Концептуальная схема (логическая) - содержит полное описание логической структуры данных, доступное СУБД (логическая схема базы данных).
3. Схема хранения (физическая) - организация хранения логических структур данных.

Язык запросов SQL - наличие описания логической структуры данных открывает возможности для выполнения достаточно сложных операций манипулирования данными внутри СУБД. Такие операции записываются на языке запросов.

Целостность и согласованность
    Ограничение целостности:
        - вес детали должен быть положительным;
        - количество знаков в телефонном номере не должно превышеть 15;
        - возраст родителей не может быть меньше возраста их биологического ребенка.
    Состояние Базы Данных, в которой выполняются пдобные условия, называются согласованными (consistent), а сами правила - условиями согласованности
    Транзакция - конечный набор операций, который переводит согласованное состояние в другое согласованное.

Отказоустойчивость
    СУБД способны обеспечить значительно более высокую надежность и доступность данных, чем надежность, или доступность оборудования, на котором эти данные хранятся и на котором работают эти систепмы.
    

--Теоретические основы БД--
Модели данных
    Модель данных - система взаимосвязанных понятий и правил, предназначенная для описания структур и свойств данных, используемых (хранимых и обрабатываемых) в информационной схеме.
        - иерархическая;
        - сетевая;
        - реляционная;
        - постреляционная;
        - многомерная;
        - объектно-ориентированная;
        - объектно-реляционная.
    В состав развитой модели данных входят:
        - способы описания данных: какие базовые типы можно использовать, каким образом строить сложные структуры данных из более простых;
        - способы описания взаимосвязей между объектами данных;
        - средства задания ограничений целостностиж
        - способы конструирования операций, которые можно использовать в рамках модели данных.

Идентификация
    Многообразие методов идентификации можно условно разделить на следующие категории:
        - по естественным признакам;
        - по искуственному значению;
        - по связи объекта.

Навигация и поиск по значениям
    Можно выделить следующие основные классы поисковых операций:
        - навигационный доступ (переходы от одного объекта к другому с помощью ссылок);
        - поиск по значениям (ассоциативный поиск).


--Реляционная модель данных (представленная в виде таблицы)--
Домен - это множество значений, обладающих некоторыми общими свойствами, то есть множество допустимых значений.
Все значения,находящиеся в доменах - скалярные (примитивные) константы (литералы).
    Примеры:
        - домен целых чисел;
        - домен длин;
        - домен текстов.

Отношения
    Фундаментальное понятие реляционной модели данных. Поэтому модель и называется реляционной.
    Пусть дана совокупность типов данных T1, T2, ..., Tn, называемых также доменами,не обязательно различных. Тогда n-арным отношением R, или отношением R степени n называют подмножество декартова произведения множеств T1, T2, ..., Tn.

Атрибуты
    Атрибут представляет свойство,которое описывает некоторую характеристику объекта.
    Именованные аргументы отношения называются атрибутами отношения.
    С каждым атрибутом связан некоторый домен.
    Множество всех атрибутов отношенич называется схемой отношения.

Кортежи
    Совокупность из n значений по одному значению (из соответствующего домена) для каждого атрибута.


https://www.postgresql.org/ - PostgreSQL - свободная объектно-реляционная СУБД.
Сильными сторонами PostgreSQL считаются:
    - высокопроизводительные и надёжные механизмы транзакций и репликации;
    - расширяемая система встроенных языков программирования: в стандартной поставке поддерживаются PL/pgSQL, PL/Perl, PL/Python и PL/Tcl; дополнительно можно использовать PL/Java, PL/PHP, PL/Py, PL/R, PL/Ruby, PL/Scheme, PL/sh и PL/V8, а также имеется поддержка загрузки модулей расширения на языке C[10];
    - наследование;
    - возможность индексирования геометрических (в частности, географических) объектов и наличие базирующегося на ней расширения PostGIS;
    - встроенная поддержка слабоструктурированных данных в формате JSON с возможностью их индексации;
    - расширяемость (возможность создавать новые типы данных, типы индексов, языки программирования, модули расширения, подключать любые внешние источники данных).


createdb name_db - утилита, которая позволяет создать базу данных name_db
createdb -O postgres name_db - change owner
createdb -U postgres name_db - change user
peer - подучает имя пользователя и проверяет соответствует ли оно имени запрашиваемой базы данных.
trust - разрешает безусловное подключение.
ll - список версий postgres.
\l - список баз данных.
\c name_db - подключиться к базе данных name_db.
\d - посмотреть на таблицы внутри базы name_db.
\q - quit.
dropdb name_db - удалить базу данных name_db.
-p - изменить порт (по умолачнию порт 5432).
